# Generic / Other Customers

Use these standard requirements when shipping to customers without a specific guide. These are conservative defaults that should be acceptable to most buyers.

<Warning>
Always ask the customer for their compliance guide first. Use these defaults only when no specific requirements are provided.
</Warning>

---

## When to Use This Guide

| Situation | Action |
|-----------|--------|
| New customer, no guide provided | Use these defaults |
| Customer says "standard packing" | Use these defaults |
| Small order, no specific instructions | Use these defaults |
| Customer has specific guide | Use their guide instead |

---

## Safe Defaults Summary

| Requirement | Safe Default |
|-------------|--------------|
| Carton Label Size | A4 (210mm × 297mm) |
| Barcode Type | Code 128 |
| Max Carton Weight | 15 kg |
| Max Carton Dimensions | 60 × 40 × 40 cm |
| Size Sorting | Single size per carton |
| Poly Bagging | Yes, individual |
| Sealing | H-seal, clear tape |
| Documents | Invoice, Packing List, COO |

### Label Specifications (Safe Defaults)

| Specification | Default |
|---------------|---------|
| Size | A4 (210mm × 297mm) |
| Orientation | Portrait |
| Paper | White, 80gsm minimum |
| Barcode Type | Code 128 |
| Barcode Height | Minimum 20mm |
| Quiet Zone | Minimum 10mm each side |
| Print | Laser printer, black ink |
| Placement | One label per carton, largest face |

<Info>
A4 is universally accepted. If customer prefers smaller, A5 (148mm × 210mm) is also widely acceptable.
</Info>

## Packing Requirements

### Carton Specifications (Safe Defaults)

| Specification | Default |
|---------------|---------|
| Maximum Weight | 15 kg gross |
| Maximum Length | 60 cm |
| Maximum Width | 40 cm |
| Maximum Height | 40 cm |
| Carton Material | Double-wall corrugated |
| Carton Color | Plain brown kraft |

<Tip>
15 kg is the safest maximum. Most warehouses can handle this weight. Heavier cartons risk rejection.
</Tip>

### Packing Rules (Safe Defaults)

<AccordionGroup>
  <Accordion title="Size Sorting">
    **Default Rule:** Single size per carton.

    This is the safest approach as most customers require it.

    | Approach | Risk Level |
    |----------|------------|
    | Single size per carton | Low risk |
    | Mixed sizes | High risk - may be rejected |

    **Exception:** Only mix sizes if customer explicitly permits it in writing.
  </Accordion>

  <Accordion title="Poly Bagging">
    **Default Rule:** Individual poly bagging required.

    | Specification | Default |
    |---------------|---------|
    | Bag Material | Clear polypropylene |
    | Bag Thickness | 25 micron minimum |
    | Closure | Self-adhesive or fold-seal |
    | Warning | Include suffocation warning |

    **Why:** Almost all customers require individual bagging. It protects garments and is expected by DCs.
  </Accordion>

  <Accordion title="Sealing Method">
    **Default Rule:** H-seal with clear tape.

    ```
    ┌───────────────┐
    │ ═══════════   │ ← Top seal
    │       │       │
    │       │       │ ← Edge seals
    │       │       │
    │ ═══════════   │ ← Bottom seal
    └───────────────┘
    ```

    | Specification | Default |
    |---------------|---------|
    | Tape Width | 50mm |
    | Tape Color | Clear |
    | Pattern | H-seal (all edges) |
    | Quality | No gaps, neat edges |

    **Why:** H-seal is the most secure and widely accepted.
  </Accordion>

  <Accordion title="Void Fill">
    **Default Rule:** Use paper void fill.

    | Material | Acceptable |
    |----------|------------|
    | Kraft paper | Yes (preferred) |
    | Tissue paper | Yes |
    | Bubble wrap | Check with customer |
    | Packing peanuts | Avoid |
    | Air pillows | Check with customer |

    **Why:** Paper is universally acceptable and environmentally friendly.
  </Accordion>
</AccordionGroup>

## Required Documents

### Standard Document Set

| Document | Copies | Required |
|----------|--------|----------|
| Commercial Invoice | 3 | Yes |
| Packing List | 2 | Yes |
| Certificate of Origin | 1 original + 2 copies | Yes |
| Bill of Lading/AWB | Original + 2 copies | Yes |

### Commercial Invoice (Safe Format)

| Field | Include |
|-------|---------|
| Your company details | Full name, address, contact |
| Customer details | Name, address |
| Invoice number | Your reference |
| Invoice date | Current date |
| PO reference | Customer's PO number |
| Description | Each style/colour |
| Quantity | Units per line |
| Unit price | Per piece |
| Total value | Line totals + grand total |
| Currency | As per PO (usually USD or GBP) |
| Incoterms | As per PO (FOB if not specified) |
| HS codes | 6-digit minimum |
| Country of origin | Pakistan |

### Packing List (Safe Format)

| Field | Include |
|-------|---------|
| Shipment reference | Your reference |
| Customer PO | Customer's PO number |
| Carton numbers | 1 of X, 2 of X, etc. |
| Carton contents | Style, colour, size, qty per carton |
| Carton weights | Gross and net |
| Carton dimensions | L × W × H |
| Totals | Total cartons, total pieces, total weight |

## Troubleshooting

<AccordionGroup>
  <Accordion title="Customer rejected shipment">
    **Cause:** Non-compliance with unknown requirements.

    **Solution:**
    1. Ask customer for specific reason
    2. Request their compliance guide
    3. Document for future shipments
    4. Update internal records
  </Accordion>

  <Accordion title="No PO reference available">
    **Cause:** Order placed informally.

    **Solution:**
    1. Request formal PO from customer
    2. If not available, use your internal reference
    3. Clearly state "No PO provided" on documents
    4. Get customer confirmation in writing
  </Accordion>

  <Accordion title="Customer wants specific barcode format">
    **Cause:** Their system requires specific format.

    **Solution:**
    1. Ask for exact specifications
    2. Request sample label for reference
    3. Test scan before bulk printing
    4. Document for future orders
  </Accordion>

  <Accordion title="Unsure about weight limit">
    **Cause:** Customer hasn't specified.

    **Solution:**
    1. Use 15 kg as safe maximum
    2. Ask customer for their limit
    3. Err on the side of lighter
    4. Record answer for future reference
  </Accordion>

  <Accordion title="Mixed sizes required by customer">
    **Cause:** Customer requested ratio packs.

    **Solution:**
    1. Get ratio in writing (e.g., 1-2-2-1)
    2. Follow exact ratio specified
    3. Label cartons clearly
    4. Document in packing list
  </Accordion>
</AccordionGroup>

## Getting Customer Requirements

### Questions to Ask

When onboarding a new customer, ask for:

| Information | Why Needed |
|-------------|------------|
| Compliance/routing guide | Full specifications |
| Label template | Exact format required |
| Barcode type | EAN-13, Code 128, etc. |
| Max carton weight | DC handling limits |
| Size sorting rule | Single or mixed |
| Required documents | What they need |
| DC address | Delivery location |
| Contact for queries | Who to ask |

### Sample Request Email

```
Subject: Shipping Compliance Requirements

Dear [Customer],

We are preparing your order [PO number] for shipment.

To ensure compliance with your DC requirements,
please provide:

1. Carton label specifications (size, format)
2. Barcode type required
3. Maximum carton weight
4. Packing requirements
5. Required shipping documents

If you have a supplier compliance guide,
please share it with us.

Thank you,
[Your name]
```
